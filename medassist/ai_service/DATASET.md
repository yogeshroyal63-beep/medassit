# MedAssist — Training Dataset

## Overview

The BERT triage model is trained on a synthetic medical symptom dataset
generated by `generate_dataset.py`. The dataset is **not committed to the
repository** because:

1. It is fully reproducible from the generation script.
2. ML dataset files (CSV, often 10–50 MB) should be stored in a model
   registry or object storage, not in Git.

## Generating the dataset

```bash
cd ai_service
python generate_dataset.py
```

This writes `dataset/triage_dataset.csv` with the following schema:

| Column    | Type   | Description                                          |
|-----------|--------|------------------------------------------------------|
| `text`    | string | Plain-language symptom description (1–4 sentences)   |
| `label`   | string | Medical condition class (CamelCase, e.g. ViralFever) |

## Dataset statistics (typical generation)

- Total samples:   ~4,800
- Train / val / test split: 85 % / 7.5 % / 7.5 % (stratified)
- Samples per class: 180–220 (approximately balanced)

## Training the model

After generating the dataset, train BioBERT:

```bash
cd ai_service
python training/train_bert.py
```

Expected metrics on the held-out test set (RTX 3050, 4 epochs):

| Metric    | Value  |
|-----------|--------|
| Accuracy  | ~88 %  |
| Macro F1  | ~0.87  |

The trained model is saved to `saved_model/final_model/` and the label
encoder to `saved_model/label_encoder.pkl`. These paths are gitignored
(see `.gitignore`) and should be stored in a model registry for production.
